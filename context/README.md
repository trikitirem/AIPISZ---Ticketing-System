# Diagramy - Opis

## Model Projektowy (`model_projektowy.puml`, `Model_Projektowy_System_Ticketingu.svg`)

Diagram pokazuje strukturę warstwową aplikacji z głównymi komponentami systemu.

Model projektowy rozbudowuje model dziedziny o dodatkowe warstwy, gdzie warstwa aplikacji dodaje przypadki użycia, a reszta warstw obsługuje kontakt ze światem zewnętrznym.

## Model Implementacyjny (`model_implementacyjny.puml`, `Model_Implementacyjny_System_Ticketingu.svg`)

Model implementacyjny rozbudowuje model projektowy o terminy związane z DDD (używaliśmy na zajęciach klasy Entity, więc poszliśmy trochę dalej i użyliśmy innych pojęć z DDD).

Diagram pokazuje pełną implementację systemu z uwzględnieniem konceptów DDD użytych w projekcie.
